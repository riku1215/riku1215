# 04: GCP 二重課金アカウントの整理

## ゴール
**月¥10,000** の二重 GCP 課金を **¥0〜¥5,000/月** に削減。

## 背景 (Gmail 分析結果)
誤って 2つの請求アカウントが並行稼働:

| お客様 ID | カード | 直近課金日 | 金額 |
|----------|--------|-----------|------|
| `0194E0-3A949C-94D9D8` (Payment profile `1054-8434-8642`) | Mastercard ****6806 | 2026/04/30 | ¥5,000 |
| `01A1B6-B52645-9E3895` | Visa ****5828 | 2026/04/19 | ¥5,000 |

## Step 1: 用途を確認

2つそれぞれのアカウントでどのプロジェクトが動いているか確認:

1. https://console.cloud.google.com/ にログイン
2. 上部のプロジェクト選択ドロップダウン → 全プロジェクト表示
3. 左メニュー **「お支払い」** → 各請求先アカウント (Billing Account) を選択
4. **「概要」** で対象プロジェクトと月次消費を確認

判断軸:
- 使っているプロジェクトがあるアカウント → 残す
- 何も動いていないアカウント → 閉じる
- 両方使っている場合 → 一方にプロジェクトを移管してから片方閉じる

## Step 2: プロジェクトの請求先アカウント移管 (両方使っている場合のみ)

1. Console → 「お支払い」→ 対象プロジェクトを選択
2. 右上 **「請求情報を変更」** → 残す側の請求先アカウントを選択
3. 確定

## Step 3: 不要な請求先アカウントを閉じる

1. Console → 「お支払い」→ 閉じる側の請求先アカウント
2. **「アカウント管理」**タブ
3. **「請求先アカウントを閉じる」** ボタン
4. 確認 → 閉鎖完了

## Step 4 (任意・推奨): 残す側で予算アラート設定

1. Console → 「お支払い」→ 残した請求先アカウント
2. **「予算とアラート」** → **「予算を作成」**
3. 月額 ¥5,000 + 80% / 100% でアラート
4. メール通知先: takaj2011311@gmail.com

## Step 5: カード情報も整理

請求先アカウントを閉じても、カード情報は残っている。不要なカード:
1. Console → 「お支払い」→ **「お支払い方法」**
2. 不要なカード行 → **「削除」**

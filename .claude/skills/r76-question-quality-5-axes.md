# R76: 質問 5 軸 構造化 (= 「目蔵 OK」 防止)

## 5 軸

| 軸 | 内容 |
|----|------|
| 1 文脈 | なぜ 聞く + 私 解釈 + 不確定要因 |
| 2 候補詳細 | a/b/c の 違い + 採用 影響 |
| 3 優先度 | ★ ranking + 即必要 / 後 OK |
| 4 影響 | 答え 別 着手 path |
| 5 緊急度 | 🔴 必須 / 🟡 推奨 / 🟢 任意 (= 削除候補) |

## 軸 5 緊急度 区分

- 🔴 **必須** = Captain 専管 (vision / 命名 / 共同者 / Public/Private)
- 🟡 **推奨** = Captain GO ある 方 が 安全 + 私 推奨 ある
- 🟢 **任意** = 私 推奨 で auto OK = **質問 削除 候補** (= R20 5 min auto fallback)

## NG / OK

- ❌ 「a/b/c」 のみ で 詳細 なし
- ❌ 「○○ で 合って いますか?」 のみ で 文脈 なし
- ❌ 全 質問 ★★★★★ (= 認知負荷 ↑)
- ❌ 緊急度 区別 なし = 全 🔴 (= 「目蔵 OK 質問」)
- ✅ 5 軸 全部 を 各 質問 で 明示
- ✅ 軸 5 🟢 = 質問 削除 + 私 推奨 で auto
